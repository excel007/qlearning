รางวัล (reward) — ปรับได้

มีผลกับก้าวถัดไปทันที — แต่ Q-table เดิมยังจำรางวัลเก่าอยู่ ถ้าอยากเห็นผลของรางวัลใหม่ชัดๆ แนะนำให้กด “ล้าง Q-table” แล้วเทรนใหม่

Bellman update — ก้าวล่าสุด

ยังไม่มีก้าว — กด “ก้าวเดียว” เพื่อเริ่ม

ตารางประวัติการเรียนรู้ — ทุกก้าวคือ transition (s, a, r, s′)

0 ก้าว
#ตอนก้าวที่ s Q(s,·) ก่อนเลือกเลือกโดย ars′ max Q(s′)targetTD error Q(s,a) เก่า → ใหม่จบตอน

แถวบนสุด = ก้าวล่าสุด · บันทึกเฉพาะโหมด “ก้าวเดียว / เล่นต่อเนื่อง” (โหมดเทรน ×100 ข้ามการบันทึกเพื่อความลื่น) · เก็บล่าสุดสูงสุด 500 ก้าว

สมการที่ใช้ และวิธีอ่าน panel

กฎอัปเดตคือ: Q(s,a) ← Q(s,a) + α · [ r + γ·max Q(s′,a′) − Q(s,a) ]

ก้อนในวงเล็บคือ TD error = (สิ่งที่ควรเป็น) − (สิ่งที่เชื่ออยู่). ถ้า error เป็นบวก แปลว่าทางนี้ดีกว่าที่คิด → ดัน Q ขึ้น; ถ้าลบ → กด Q ลง. ขนาดที่ขยับถูกคูณด้วย α

รางวัลเริ่มต้น: ถึงเป้า +10, ตกหลุม −10, แต่ละก้าว −0.1 (ลงโทษเล็กน้อยเพื่อบีบให้หาเส้นทางสั้นที่สุด), ชนกำแพง −0.1 — ทั้งหมดปรับได้ที่การ์ด “รางวัล”

ลองออกแบบรางวัล (reward shaping): ตั้งค่าก้าวเป็น 0 แล้วเทรน — เอเจนต์จะไม่รีบ เพราะการเดินวนไม่เสียอะไร เส้นทางที่ได้อาจอ้อม. เพิ่มโทษชนกำแพงเป็น −1 จะเห็นว่ามันเรียนรู้ หลบกำแพงเร็วขึ้น. หรือลองทำให้ตกหลุมเป็น +5 (รางวัลบวก!) เอเจนต์จะ “วิ่งหาหลุม” แทนเป้า — ตัวอย่างคลาสสิกว่ารางวัลที่ออกแบบผิดทำให้ AI เรียนรู้พฤติกรรมที่เราไม่ต้องการ

ทดลองดู: ตั้ง γ=0 แล้วเทรน — เอเจนต์จะ “สายตาสั้น” มองไม่เห็นเป้าจนกว่าจะอยู่ติดมัน เพราะมันไม่สนรางวัลอนาคต. เพิ่ม γ กลับเป็น 0.95 จะเห็นค่า Q “ไหล” ย้อนจากเป้ามาทั่วทั้งกริด
